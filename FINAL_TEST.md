# ✅ 최종 테스트 가이드 (파란 화면 해결!)

## 🎯 문제 해결 완료!

### "파란 화면만 나와요" → ✅ 완전 해결!

**변경 사항**:
1. ✅ Guest 모드 추가 (즉시 플레이 가능)
2. ✅ 게임 상태 초기화 개선
3. ✅ 덱이 비어도 정상 작동
4. ✅ 디버그 정보 표시 (개발용)
5. ✅ 모든 화면 렌더링 보장

---

## 🚀 지금 바로 테스트!

### STEP 1: 완전 정리

```powershell
# 모든 Node 프로세스 종료
taskkill /IM node.exe /F
```

### STEP 2: 실행

```powershell
# 프론트엔드 + 백엔드 동시 실행
npm run dev:full
```

### STEP 3: 브라우저 접속

```
http://localhost:5173
```

## 🎮 화면별 작동 테스트

### 1. 로그인 화면 ✅

**표시되는 것들**:
- 💳 Financial Card Storm (타이틀)
- 은행 거래 데이터 기반 카드 배틀 게임 (설명)
- 이름 입력 칸
- 비밀번호 입력 칸
- "로그인" 버튼
- "신규 고객입니다" 버튼
- **"🎮 Guest로 시작하기"** 버튼 ← NEW!
- 테스트 계정 안내

**테스트**:
```
"🎮 Guest로 시작하기" 클릭
↓
메인 화면으로 이동 ✅
```

### 2. 메인 화면 ✅

**표시되는 것들**:
- Financial Card Storm (헤더)
- Guest 체험 모드 또는 [이름]님의 거래 패턴 (부제목)
- "내 거래 내역" 버튼
- "회원가입하기" (Guest) 또는 "로그아웃" (회원)
- **디버그 정보** (노란색 박스) ← NEW!
  - 현재 화면, 사용자, 거래 건수, 카드 수 표시
- "당신의 소비 패턴으로 보스를 쓰러뜨리세요" (제목)
- 거래 200건, 생성된 카드 N장 (설명)
- **"싱글 플레이 시작"** 버튼
- **"멀티플레이 (PvP)"** 버튼
- **"거래 데이터 미리보기"** 버튼

**테스트**:
```
"싱글 플레이 시작" 클릭
↓
전투 화면으로 이동 ✅
```

### 3. 전투 화면 (싱글 플레이) ✅

**표시되는 것들**:
- 상단: 소비 패턴 보스 (HP 20/20, 방어막, 상태 효과)
- 중앙: 전장 영역 + 전투 로그
- 우측: 에너지 바 (0~10) + 턴 종료 버튼
- 하단: 플레이어 (HP 20/20) + 손패 카드 3~5장

**카드 UI**:
- 비용 배지 (좌측 상단)
- 카드 이름
- 카테고리 태그
- 카드 이미지 (이모지)
- 효과 아이콘 (DAMAGE, HEAL 등)
- 설명 텍스트
- 공격력/방어력 (하단)

**테스트**:
```
손패에서 카드 클릭
↓
보스 HP 감소 ✅
↓
"턴 종료" 클릭
↓
보스 반격 → 내 HP 감소 ✅
↓
다음 턴 시작 (에너지 +1, 카드 1장 드로우) ✅
```

### 4. 멀티플레이 로비 ✅

**표시되는 것들**:
- 멀티플레이 로비 (타이틀)
- 🟢 서버 연결됨 (녹색) 또는 🔴 서버 연결 안됨 (빨간색)
- 닉네임 입력
- 방 이름 입력
- "방 생성" 버튼
- 대기 중인 방 목록

**테스트**:
```
닉네임 입력: 플레이어1
방 이름 입력: 테스트방
↓
"방 생성" 클릭
↓
대기실로 이동 ✅
```

## 🐛 만약 여전히 파란 화면이면?

### 체크 1: 디버그 정보 확인

메인 화면 상단에 **노란색 박스**가 보여야 합니다:

```
디버그: 현재 화면 = MAIN | 사용자 = Guest | 거래 = 200건 | 카드 = 25장 | 덱 = 25장
```

### 체크 2: 브라우저 콘솔 확인

F12 누르고 Console 탭 확인:
- 에러 메시지 있는지 확인
- React 에러 있는지 확인

### 체크 3: 캐시 삭제

```powershell
# Ctrl + Shift + R (강제 새로고침)
```

또는

```powershell
# 브라우저 캐시 완전 삭제 후 재접속
```

### 체크 4: 완전 재시작

```powershell
# 1. 모든 프로세스 종료
taskkill /IM node.exe /F

# 2. 캐시 정리
npm cache clean --force

# 3. 재설치
Remove-Item -Recurse -Force node_modules
npm install

# 4. 실행
npm run dev:full
```

## 💡 정상 작동 확인 방법

### 로그인 화면 (첫 화면)

```
✅ 타이틀 보임
✅ 입력 칸 보임
✅ 버튼 3개 보임 (로그인, 신규 고객, Guest)
```

### 메인 화면 (Guest 로그인 후)

```
✅ 헤더 보임 (Financial Card Storm)
✅ 디버그 정보 보임 (노란색 박스)
✅ 중앙에 설명 텍스트 보임
✅ 버튼 3개 보임 (싱글, 멀티, 거래 내역)
```

### 전투 화면 (싱글 플레이 클릭 후)

```
✅ 보스 패널 보임 (상단)
✅ 전장 영역 보임 (중앙)
✅ 에너지 바 보임 (우측)
✅ 손패 카드 보임 (하단, 3~5장)
✅ 카드 클릭 가능
✅ 턴 종료 버튼 작동
```

## 📸 각 화면 스크린샷 체크리스트

### 로그인 화면

- [ ] 배경색: 어두운 그라데이션
- [ ] 중앙 박스: 파란색 테두리
- [ ] 버튼: 파란색 (Guest는 점선 테두리)

### 메인 화면

- [ ] 배경색: 어두운 그라데이션
- [ ] 헤더: 상단 고정
- [ ] 중앙: 큰 설명 텍스트 + 버튼 3개
- [ ] 디버그 정보: 노란색 박스

### 전투 화면

- [ ] 상단: 보스 (빨간색 테두리)
- [ ] 중앙: 전장 + 로그
- [ ] 우측: 에너지 바 (세로)
- [ ] 하단: 플레이어 + 카드 (5장)
